

 1. What is SSH

**SSH (Secure Shell)** is a protocol used to connect to remote devices securely. All communication is encrypted, so data cannot be intercepted.

2. Uses of SSH

. Connect to a remote server:

. Manage servers and execute commands as if you are directly on the machine.
. Securely transfer files between your local machine and the server.
. Use with GitHub/Git for pushing or pulling code without entering a password each time.

 3. Example with GitHub

''''
git clone git@github.com:username/project.git
''''

This uses your SSH Key for authentication automatically.

### 4. Setting up SSH Key for GitHub

1. Generate an SSH key on your local machine:

```bash
ssh-keygen -t ed25519 -C "your_email@example.com"
```

Press Enter for default options or specify a custom location.

2. Display the public key:

```bash
cat ~/.ssh/id_ed25519.pub
```

3. Copy the key and add it to GitHub:

* Go to GitHub → Settings → SSH and GPG keys → New SSH Key → Paste your key.

4. Clone repositories using SSH:

```bash
git clone git@github.com:username/project.git
```

 Summary of Git Workflow with SSH

1. Clone the repository using SSH.
2. Make changes to your project.
3. Stage the changes:

```bash
git add .
```

4. Commit the changes:

```bash
git commit -m "Describe your changes"
```

5. Push changes to GitHub:

```bash
git push origin main
```
