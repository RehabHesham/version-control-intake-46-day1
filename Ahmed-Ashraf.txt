SSH is a cryptographic network protocol used for secure operating services over an unsecured network, primarily for remote login and command-line execution. It was designed by Tatu Ylönen in 1995 as a secure replacement for unprotected protocols like Telnet and FTP. The protocol relies on public-key cryptography to authenticate the remote computer and the user. It consists of three main layers: the Transport Layer for encryption, the User Authentication Layer, and the Connection Layer for multiplexing channels.

In a typical setup, a user generates a key pair: a private key kept locally and a public key stored on the server in ~/.ssh/authorized_keys. To manage these keys on Unix-like systems or PowerShell, common commands include ssh-keygen for creating keys and ls -al ~/.ssh for listing existing ones. This method allows for automated, passwordless logins while maintaining high security. Beyond simple shells, SSH supports tunneling, port forwarding, and secure file transfers via SFTP or SCP. Today, OpenSSH is the most widely used implementation, supporting the modern SSH-2 standard.
